Контейнер PoS/PoW останавливается через минуту после запуска
Часть пользователей сталкивается с трудностями при запуске PoS/PoW на определенных процессорах (обычно старых). Причина в несоответствии поставляемой в Docker-образах конфигурации RandomX. Например, для RandomX опеределенно предпочтительна аппаратная поддержка AES. Мы не стали искать точные требования для работы RandomX в поставляемых образах, поскольку это очень сложная задача (Вы можете обратиться к разработчикам RandomX если хотите углубиться в нее). Вместо этого мы предоставляем инструкцию как собрать RandomX из исходного кода на вашем компьютере. Этот метод прекрасно сработал для нескольких владельцев PoS узлов. Но может не подойти для именно вашего CPU, поэтому поделитесь через pull request GitHub своими трюками, как заставить RandomX работать.

Предварительные требования: операционная система на основе Linux и установленные git, NodeJS, C/C++ developer tools. Ищите в интернете иструкции по установке для вашего дистрибутива Linux. Например инстурменты разработчика будут в метапакете "build-essentials" в Ubuntu OS и группа пакетов "base-devel" в Arch Linux.
